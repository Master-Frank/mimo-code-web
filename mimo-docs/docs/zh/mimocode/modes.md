# 工作模式

MiMo Code 提供多种工作模式，适应不同的开发场景。

---

## Build 模式

默认模式。AI 可以直接读写文件、执行命令，适合日常开发任务。

```bash
<TAB>
```

在 Build 模式下，MiMo Code 可以：
- 读取和修改文件
- 执行 shell 命令
- 运行测试
- 创建新文件

---

## Plan 模式

计划模式下，AI 只会输出行动计划而不修改文件，适合复杂或高风险任务。

```bash
<TAB>
```

在 Plan 模式下，MiMo Code 会：
- 分析代码库
- 提出改动建议
- 列出需要修改的文件
- 不执行任何实际修改

> **提示：** 在 Plan 模式下确认计划后，切换回 Build 模式让 AI 实施。

---

## 自定义模式

你可以在配置文件中定义自定义工作模式，指定不同的工具权限和行为。

```json
{
  "modes": {
    "review": {
      "allowedTools": ["read", "grep"],
      "description": "只读审查模式"
    }
  }
}
```
